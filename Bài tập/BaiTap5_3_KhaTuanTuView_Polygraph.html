Lời giải Bài tập 5.3 (Giáo trình)

Xét tính khả tuần tự View thông qua Đồ thị phức G(Sk)

⚙️ Quy tắc phân tích Đồ thị phức Polygraph G(S) • Cung cố định (liền nét): Ti → Tj nếu Tj đọc dữ liệu từ Ti (wi(X) trước rj(X)). Ghi cuối cùng dẫn về đỉnh kết thúc Tf.
• Cung lựa chọn (đứt nét): Nếu có cung cố định Ti → Tj trên mục dữ liệu X, và có một giao tác Tk khác có thao tác wk(X), ta phải chọn một trong hai hướng: (Tk → Ti) hoặc (Tj → Tk) sao cho đồ thị cuối cùng không tạo thành chu trình đóng.
S₁ = { w₁(A), r₃(A), r₂(A), w₂(A), r₁(A), w₃(A) } Không khả tuần tự View

1. Phân tích quan hệ đọc dữ liệu từ đâu (Cung cố định)

  • w₁(A) được đọc bởi r₃(A) và r₂(A) → Có cung cố định: T₁ → T₃ và T₁ → T₂
  • w₂(A) được đọc bởi r₁(A) → Có cung cố định: T₂ → T₁
  • Hành động ghi sau cùng trên mục dữ liệu A thuộc về w₃(A) của T₃ → Có cung dẫn về đỉnh kết thúc: T₃ → Tf

2. Tập hợp đỉnh và cung biểu diễn đồ thị G(S₁)

Đỉnh: V = { T₁, T₂, T₃ }
Cung cố định: E_fixed = { (T₁ → T₃), (T₁ → T₂), (T₂ → T₁) }
Cung lựa chọn tương tranh bởi T₃: { (T₃ → T₂) hoặc (T₁ → T₃) }
❌ Kết luận: Ngay từ tập cung cố định, đồ thị đã tồn tại một chu trình đóng kín bất biến song hướng nối giữa hai đỉnh (T₁ ⇄ T₂). Do đó, không có cách chọn hướng cung lựa chọn nào để phá bỏ chu trình.
👉 Lịch biểu S₁ KHÔNG KHẢ TUẦN TỰ VIEW.
S₂ = { w₁(A), r₂(A), w₂(A), r₁(A) } Không khả tuần tự View

1. Phân tích quan hệ đọc dữ liệu từ đâu (Cung cố định)

  • r₂(A) nhận dữ liệu do w₁(A) ghi ra → Tạo cung cố định: T₁ → T₂
  • r₁(A) nhận dữ liệu do w₂(A) ghi ra → Tạo cung cố định: T₂ → T₁
  • Giao tác ghi sau cùng lên mục dữ liệu A là T₂ (w₂(A)) → Tạo cung: T₂ → Tf

2. Tập hợp đỉnh và cung biểu diễn đồ thị G(S₂)

Đỉnh: V = { T₁, T₂, }
Cung cố định: E_fixed = { (T₁ → T₂), (T₂ → T₁) }
❌ Kết luận: Xuất hiện chu trình khép kín nối trực tiếp giữa hai đỉnh (T₁ ⇄ T₂). Đồ thị chứa chu trình.
👉 Lịch biểu S₂ KHÔNG KHẢ TUẦN TỰ VIEW.
S₃ = { w₁(A), r₂(A), w₃(A), r₄(A), w5(A), r₆(A) } Khả tuần tự View

1. Phân tích quan hệ đọc dữ liệu từ đâu (Cung cố định)

  • r₂(A) đọc từ w₁(A) → Có cung cố định: T₁ → T₂
  • r₄(A) đọc từ w₃(A) → Có cung cố định: T₃ → T₄
  • r₆(A) đọc từ w₅(A) → Có cung cố định: T₅ → T₆
  • Ghi sau cùng lên mục dữ liệu A là w₅(A) của T₅ → Có cung: T₅ → Tf

2. Lựa chọn hướng cho các cung tương tranh (Cung đứt nét)

  • Xét cung cố định T₁ → T₂: Có các giao tác ghi chồng khác là T₃, T₅. Ta chọn hướng đứng sau T₂ để không ngắt mạch luồng đọc: Chọn hướng T₂ → T₃ và T₂ → T₅.
  • Xét cung cố định T₃ → T₄: Có giao tác ghi chồng T₅. Ta chọn hướng đứng sau T₄: Chọn hướng T₄ → T₅.

3. Kết quả đồ thị hoàn chỉnh sau sắp xếp Topo

Đồ thị dạng đường đi thẳng: T₁ → T₂ → T₃ → T₄ → T₅ → T₆
✅ Kết luận: Đồ thị hoàn toàn không chứa chu trình tuần hoàn.
👉 Lịch biểu S₃ KHẢ TUẦN TỰ VIEW và tương đương với lịch biểu tuần tự: S' = (T₁, T₂, T₃, T₄, T₅, T₆).
S₄ = { r₂(B), w₂(A), r₁(A), r₃(A), w₁(B), w₂(B), w₃(B) } Khả tuần tự View

1. Phân tích quan hệ đọc dữ liệu từ đâu (Cung cố định)

  • Mục dữ liệu A: r₁(A) và r₃(A) đều nhận dữ liệu do w₂(A) ghi ra → Tạo cung cố định: T₂ → T₁ và T₂ → T₃.
  • Mục dữ liệu B: r₂(B) đọc dữ liệu ban đầu từ hệ thống (trước khi các lệnh ghi xuất hiện) → Coi như đọc từ giao tác khởi tạo hệ thống: T₀ → T₂.
  • Hành động ghi cuối cùng lên B thuộc về w₃(B) của T₃ → Tạo cung: T₃ → Tf.

2. Lựa chọn hướng cho các cung tương tranh dựa trên dữ liệu B

  • Vì T₂ phải đọc dữ liệu ban đầu trên B, các giao tác ghi chồng dữ liệu B (w₁(B), w₃(B)) buộc phải được xếp thực hiện hoàn toàn phía sau thao tác đọc của T₂ để tránh làm nhiễm bẩn dữ liệu gốc.
  • Ta chọn các cung hướng đi ra khỏi T₂: Chọn hướng T₂ → T₁ và T₂ → T₃.
  • Dựa trên trình tự ghi chồng mục B trong lịch biểu gốc: w₁(B) chạy trước w₃(B) → Chọn thêm hướng cung: T₁ → T₃.

3. Kết quả đồ thị hoàn chỉnh sau sắp xếp Topo

Đồ thị đạt được thứ tự Topo vô chu trình: T₂ → T₁ → T₃
✅ Kết luận: Đồ thị hoàn toàn không chứa chu trình tuần hoàn.
👉 Lịch biểu S₄ KHẢ TUẦN TỰ VIEW và tương đương với lịch biểu tuần tự: S' = (T₂, T₁, T₃).
