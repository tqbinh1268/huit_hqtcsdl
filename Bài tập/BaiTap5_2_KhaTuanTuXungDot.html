Bài tập 5.2 - Xét tính khả tuần tự xung đột

Môn học: Hệ Quản Trị Cơ Sở Dữ Liệu

💡 Quy tắc xét đồ thị trình tự (Precedence Graph) Hai thao tác thuộc hai giao tác khác nhau xung đột nếu cùng thao tác trên một mục dữ liệu và có ít nhất một thao tác là Ghi (Write). Thứ tự thao tác trước/sau quyết định hướng mũi tên từ Ti → Tj. Nếu đồ thị không chứa chu trình thì lịch biểu đó đạt tính khả tuần tự xung đột.
Lịch biểu S₁ = { w₁(A), r₂(A), r₃(A), w₄(A) } Khả tuần tự

1. Phân tích các cặp xung đột trên dữ liệu A

  • w₁(A) xuất hiện trước và xung đột với r₂(A) → Tạo cung: T₁ → T₂
  • w₁(A) xuất hiện trước và xung đột với r₃(A) → Tạo cung: T₁ → T₃
  • w₁(A) xuất hiện trước và xung đột với w₄(A) → Tạo cung: T₁ → T₄
  • r₂(A) xuất hiện trước và xung đột với w₄(A) → Tạo cung: T₂ → T₄
  • r₃(A) xuất hiện trước và xung đột với w₄(A) → Tạo cung: T₃ → T₄

2. Đồ thị trình tự P(S₁)

Tập đỉnh: V = { T₁, T₂, T₃, T₄ }
Tập cung: E = { (T₁ → T₂), (T₁ → T₃), (T₁ → T₄), (T₂ → T₄), (T₃ → T₄) }
👉 Kết luận: Đồ thị không có chu trình. Lịch biểu S₁ KHẢ TUẦN TỰ XUNG ĐỘT.
Thứ tự tuần tự tương đương: (T₁, T₂, T₃, T₄) hoặc (T₁, T₃, T₂, T₄).
Lịch biểu S₂ = { r₁(A), w₂(A), w₁(A), w₃(A), r₂(A) } Không khả tuần tự

1. Phân tích các cặp xung đột trên dữ liệu A

  • r₁(A) xuất hiện trước và xung đột với w₂(A) → Tạo cung: T₁ → T₂
  • w₂(A) xuất hiện trước và xung đột với w₁(A) → Tạo cung: T₂ → T₁ (Phát hiện chu trình)
  • w₁(A) xuất hiện trước và xung đột với w₃(A) → Tạo cung: T₁ → T₃
  • w₃(A) xuất hiện trước và xung đột với r₂(A) → Tạo cung: T₃ → T₂

2. Đồ thị trình tự P(S₂)

Tập đỉnh: V = { T₁, T₂, T₃ }
Tập cung: E = { (T₁ → T₂), (T₂ → T₁), (T₁ → T₃), (T₃ → T₂) }
👉 Kết luận: Đồ thị xuất hiện chu trình đóng giữa hai đỉnh (T₁ ⇄ T₂). Lịch biểu S₂ KHÔNG KHẢ TUẦN TỰ XUNG ĐỘT.
Lịch biểu S₃ = { r₂(B), w₂(A), r₁(A), r₃(A), w₁(B), w₂(B), w₃(B) } Không khả tuần tự

1. Phân tích các cặp xung đột

  • Mục dữ liệu A: w₂(A) trước r₁(A), r₃(A) → Tạo cung: T₂ → T₁ và T₂ → T₃
  • Mục dữ liệu B: r₂(B) trước w₁(B), w₃(B) → Tạo cung: T₂ → T₁ và T₂ → T₃
  • Mục dữ liệu B: w₁(B) trước w₂(B) → Tạo cung: T₁ → T₂ (Phát hiện chu trình)
  • Mục dữ liệu B: w₂(B) trước w₃(B) → Tạo cung: T₂ → T₃

2. Đồ thị trình tự P(S₃)

Tập đỉnh: V = { T₁, T₂, T₃ }
Tập cung: E = { (T₂ → T₁), (T₂ → T₃), (T₁ → T₂) }
👉 Kết luận: Đồ thị chứa chu trình tương tranh tương hỗ giữa (T₁ ⇄ T₂). Lịch biểu S₃ KHÔNG KHẢ TUẦN TỰ XUNG ĐỘT.
Lịch biểu S₄ = { r₂(A), r₁(B), w₂(A), r₃(A), w₁(B), r₂(B), w₂(B) } Khả tuần tự

1. Phân tích các cặp xung đột

  • Mục dữ liệu A: w₂(A) xuất hiện trước và xung đột với r₃(A) → Tạo cung: T₂ → T₃
  • Mục dữ liệu B: r₁(B) xuất hiện trước và xung đột với w₂(B) → Tạo cung: T₁ → T₂
  • Mục dữ liệu B: w₁(B) xuất hiện trước và xung đột với r₂(B), w₂(B) → Tạo cung: T₁ → T₂

2. Đồ thị trình tự P(S₄)

Tập đỉnh: V = { T₁, T₂, T₃ }
Tập cung: E = { (T₁ → T₂), (T₂ → T₃) }
👉 Kết luận: Đồ thị là một đường đi thẳng vô chu trình theo hướng T₁ → T₂ → T₃. Lịch biểu S₄ KHẢ TUẦN TỰ XUNG ĐỘT.
Thứ tự tuần tự tương đương duy nhất: (T₁, T₂, T₃).
